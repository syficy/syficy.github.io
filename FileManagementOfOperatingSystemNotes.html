操作系统笔记之文件管理
Created 2020-12-02| Updated 2021-01-06|操作系统
Post View:

前言

本文是操作系统文件管理的学习笔记

Overview: 

1. File Concept
2. Access Methods
3. Directory Structure
4. File System Mounting
5. File Sharing
6. Protection

概念

File Concept

文件是操作系统创建的逻辑上的存储单元，
文件属性：
1. Identifier
2. Name
3. Type
4. Location
5. Size
6. Protection
7. Last-access time and Last-updated time
文件操作：
1. create
2. write
3. read
4. reposition with a file
5. deleting a file
6. truncating a file

Directory

目录操作:
1. search for a file
2. create a file
3. delete a file
4. list a directory
5. rename a file
6. traverse the file system

Single-Level Directory: filename has to be unique and poor efficiency
Two-Level Directory: a separate dir for each user, and single-level dir problems still exists per user
Tree-Structured Directory: Have concept of Absolute path and Relative path
Acyclic-Graph Directory: Use links to share files or directories, use reference counters to record, if the counters is 0,delete the file.

关于文件的权限处理： Access-Control List, 用3个类来管理每个程序， 3ACL（RWX),
owner: (eg. 7 = RWX = 111)
group: (eg. 6 = RWX = 110)
public(others): (eg. 4 = RWX = 100)
