操作系统笔记之进程管理
Created 2020-12-02| Updated 2020-12-04|操作系统
Post View:

前言

进程管理是操作系统的重点和难点。啃书莫谓难。

Process概念

Program： passive entity, binary stored in disk
Process: active entity: a program in execution in memory

A process includes:
1. Code segment （text section）
2. Data section
3. Stack
4. heap
5. Current activity(Program counter, register contents)
6. A set of associated resources

Treeds（线程）:
线程是CPU处理的最基本单位，所有的线程属于同一进程共享，共享以下内容:

  1. Code Section
  2. Data Section
  3. OS resouces （eg: open files and signals）

进程状态:

  1. New: the process is being created
  2. Ready: the precess is in the memory waiting to be assigned to a processor
  3. Running: instructions are abeing executed by CPU
  4. Waiting：the process is waiting for events to occur
  5. Terminated: the process has finished execution

进程状态转换图:

两种常见的进程状态：
running-> waiting : IO or enent wait (like sleep)
scheduler: select ready to running

Content switch: save the old preocess and reload the new process

进程调用队列，进程可能存在于以下的队列中：
job queue： set of all processes
ready queue: ready and waiting to execute
devices queue: waiting for an IO devices

CPU sheduler

short-term: ready -> run, memory -> cpu
long-term: new -> ready, disk -> memory
medium-term: ready -> wait

在操作系统的process是一个进程树，通过父进程创建子进程

Deadlock Problm

死锁是一个进程占有了一些资源由需要另外的资源，而另外的资源被别的进程所占有，从而造成的互相等待锁死的现象。

necessary conditions

死锁有四大必要条件：

  1. Mutual exclusion resource不可共享，只能一个程序同时使用
  2. Hold & Wait 已hold了一些resource并且wait另外一些resource
  3. No preemption resouces只能被占有程序主动释放
  4. Circular wait: 存在resouces循环等待

resouce-allocation graph

资源类型用R1、R2表示
每种资源的不同数量用instance表示
程序对资源调用的周期是：request -> use -> release

从资源指向进程的箭头表示资源被占用
从进程指向资源的箭头表示进程需要资源
